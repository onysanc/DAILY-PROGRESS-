      🔄 WOPS-Outbound Auto-Check Approval

      Controls whether Start Pick / End Pick / Pulling / Checking suggestions pulled live from WOPS-Outbound need someone to click Apply (Outbound or Admin, via the WOPS Sync panel), or whether they get checked off automatically the moment they're detected.

      Off (default): matches wait in the WOPS Sync panel for a user with the "Apply WOPS-Outbound Sync" permission to approve. On: matches write straight to the dispatch as soon as WOPS-Outbound shows them, for every user viewing the app — no click needed.

      ⏸ Break Times

      Recurring daily windows (e.g. AM Mid Break 10:00–10:30 AM, AM Lunch Break 12:00–1:00 PM) that are automatically excluded from every time-based computation app-wide — step durations, delay/gap tracking, pending/stuck alerts, leaderboards, throughput, and ETA. Applies going forward and recalculates existing records live.

      ⏱ Delay / Gap Log

      ℹ
      Measures the time gap between every consecutive milestone (Start Pick → End Pick → PGI → Invoiced → DS Printed → Checking → Loaded) for each dispatch. A gap is flagged as an incident when it exceeds the threshold set below. Thresholds are admin-adjustable and saved for everyone. Use Full Records to view/extract every timestamp and step duration for each dispatch in one row, or click Details on any row for the complete record. Admins can exclude a specific dispatch from all Delay/Gap stats (Comp. column, or from the Details view) — a written reason is required, and the dispatch stays visible in the log tagged Excluded.
      ⚙ Delay/Gap incident thresholds are now configured under User Management → Alerts & Notification Settings.
